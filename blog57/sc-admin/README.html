Adjust “sc-admin/lib/settings.php” to reflect your settings (line 5 and 6 generally)

Overview:

All content is stored as IDs. IDs can be large blocks of copy, nav elements, headers, footers, or even templates. You can nest IDs within other IDs up to 15 levels deep.

The page is generated when an ID is called which instructs to assembler script to load in the template specified in the ID and swap out content in the template with content unique to the ID. If no template is supplied, the Main code in the ID is displayed and the other fields are ignored.

Templates: IDs can also act as templates. They must contain the $slvr_page tag where you want the body of the child ID to appear. to specify a template for an ID, enter the ID of the template with no dollar sign.

File structure:

IDs are plain text files that live in the “IDs” folder.

You can make a back up of the existing IDs by clicking the make back up feature in the admin page

You can also use IDs as templates.

IDs can contain links to other IDs as well as templates

Use an ID as an include inside another ID or template by indicating it with a double dollar sign, example: $$footercopy

URL REWRITING:

A .htaccess file is included that will rewrite URLS in the home directory from /?p=about to /about 
This allows you to link to other IDs just by using their name and not having to specify ?p=ID
Depending on your server configuration you may need to adjust this file
The main replace pattern is RewriteRule ^([^/][a-zA-Z0-9-]+)$ index.php?p=$1 [L]

Tips:

Avoid special characters and underscores in IDs as they will be converted to dashes (multiple dashes are condensed to single dashes)

The double dollar sign($$) is a special token and is parsed out to signify the inclusion of an ID
